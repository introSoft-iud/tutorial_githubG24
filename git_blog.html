Tutorial de Git y GitHub

Introducción a Git

Git es un sistema de control de versiones distribuido que permite gestionar el código fuente y colaborar en proyectos de manera eficiente. A través de comandos, Git rastrea los cambios, almacena versiones anteriores y facilita la colaboración.

Configuración Inicial de Git

Antes de empezar a usar Git, es necesario configurarlo con tu nombre y correo electrónico para que estos datos aparezcan en cada uno de tus commits.


    git config --global user.name "Tu Nombre"
    git config --global user.email "tu-email@ejemplo.com"

Inicializar un Repositorio Git

Para comenzar a usar Git en un proyecto, debes inicializar un repositorio en el directorio del proyecto:


    git init

Agregar Archivos al Staging (git add)

Cuando realizas cambios en tu proyecto, Git los rastrea, pero necesitas agregarlos al "staging" antes de confirmarlos:


    git add nombre-del-archivo

O para agregar todos los archivos modificados:


    git add .

Guardar Cambios (commit)

Después de agregar los archivos al "staging", debes confirmar los cambios, creando un commit:


    git commit -m "Mensaje descriptivo del cambio"

Crear y Cambiar entre Ramas

Una de las características más útiles de Git es la capacidad de trabajar en diferentes ramas. Para crear una nueva rama y cambiar a ella:


    git checkout -b nombre-de-la-rama

Para cambiar entre ramas existentes:


    git checkout nombre-de-la-rama

Fusionar Ramas (merge)

Para fusionar los cambios de una rama a la rama actual, usa el siguiente comando:


    git merge nombre-de-la-rama

Ver el Estado del Repositorio (git status)

El comando git status te permite ver el estado actual del repositorio, mostrando qué archivos han sido modificados y cuáles están en el área de staging.


    git status

Introducción a GitHub

GitHub es una plataforma basada en la web que permite a los desarrolladores alojar y gestionar repositorios de Git. Ofrece control de versiones en la nube, colaboración, y una comunidad activa para trabajar en proyectos de código abierto.

Sincronizar tu Repositorio Local con GitHub

Una vez que tienes un repositorio local, puedes sincronizarlo con un repositorio remoto en GitHub.

  1. Crea un nuevo repositorio en GitHub.
  2. Conecta tu repositorio local al remoto con el siguiente comando: 
    
        git remote add origin https://github.com/tu-usuario/nombre-del-repositorio.git
  3. Sube los cambios al repositorio remoto: 
    
        git push -u origin main

Generar y Configurar una Llave SSH para GitHub

Para evitar tener que ingresar tus credenciales cada vez que empujes cambios, puedes configurar una llave SSH para autenticarte de manera segura con GitHub.

  1. Genera una nueva llave SSH: 
    
        ssh-keygen -t ed25519 -C "tuemail@ejemplo.com"
  2. Añade la llave SSH generada al agente SSH: 
    
        eval "$(ssh-agent -s)"
        ssh-add ~/.ssh/id_ed25519
  3. Copia la llave pública: 
    
        cat ~/.ssh/id_ed25519.pub
  4. Añádela a GitHub en la sección de Settings > SSH and GPG keys.
  5. Prueba la conexión: 
    
        ssh -T git@github.com

¿Qué sucede si usas un correo diferente al de GitHub?

Si usas un correo que no está asociado a tu cuenta de GitHub, los commits no estarán vinculados a tu perfil. Para evitar esto, asegúrate de configurar el correo correcto o añade el correo alternativo a tu cuenta de GitHub.
